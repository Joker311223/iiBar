使用指南

从安装到第一套布局

下载 DMG 即可安装 iiBar 2.0.0；无需安装 Xcode。

安装

直接下载 iiBar.dmg

  1. 打开下载的 DMG。
  2. 将 iiBar 拖入“应用程序”文件夹。
  3. 从“应用程序”启动 iiBar。
首次打开提示

当前公开安装包采用 ad-hoc 签名。如果 macOS 阻止启动，请前往“系统设置 → 隐私与安全性”，在 iiBar 提示旁选择“仍要打开”。

从源码构建

开发者可准备 macOS 26+ 和 Xcode 26+，然后执行：

git clone git@github.com:Joker311223/iiBar.git
cd iiBar
open iiBar.xcodeproj

在 Xcode 顶部选择 iiBar Scheme 与 My Mac 目标，点击运行。若只需验证编译，可关闭代码签名后执行仓库 README 中的 xcodebuild 命令。

权限

必要权限

辅助功能用于识别和移动菜单栏项目；屏幕录制用于布局缩略图、工具提示和部分浮层能力。

首次启动时按照引导授权。也可以稍后前往“iiBar 设置 → 高级 → 权限”，跳转到 macOS“隐私与安全性”页面。

整理布局

  1. 打开“设置 → 菜单栏布局”。
  2. 将常用项目保留在“可见”区域。
  3. 将偶尔使用的项目拖到“隐藏”区域。
  4. 将无需直接访问的项目拖到“始终隐藏”区域。
  5. 在“常规”中配置悬停、点击、滚动或手势触发方式。

如果菜单栏空间不足，可开启 iiBar Bar，让隐藏项目显示在菜单栏下方的独立面板中。

布局档案

档案会保存当前区域和项目顺序。你可以为办公、演示或外接显示器场景准备不同档案，并将档案绑定到前台应用或专注模式。

建议

首次保存档案前，先确认所有需要管理的菜单栏应用都已启动，以免档案缺少项目。

问题排查

项目没有出现在布局列表

确认辅助功能权限已经授予，退出并重新启动对应菜单栏应用，然后重新打开布局设置。

系统反复请求权限

在“系统设置 → 隐私与安全性”中移除 iiBar，重新启动应用并再次授权。

构建失败

确认当前使用完整 Xcode，而不是仅安装 Command Line Tools；同时检查 Xcode 与 macOS SDK 是否满足项目要求。
